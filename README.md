# NexusProcure End-to-End (E2E) Test Suite

This directory contains the automated End-to-End (E2E) test suite for the **NexusProcure** procurement tracking system. The test suite is built using **Playwright** and validates key user scenarios, access controls, business rules, and form validation flows.

---

## Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- npm (comes bundled with Node.js)

### Installation
1. Navigate to the `e2e-tests` directory:
   ```bash
   cd e2e-tests
   ```
2. Install the package dependencies:
   ```bash
   npm install
   ```
3. Install the required Playwright browser binaries (Chromium):
   ```bash
   npx playwright install chromium
   ```

---

## Running Tests

All execution commands should be run from inside the `e2e-tests` directory.

| Action | Command | Description |
|---|---|---|
| **Run All Tests** | `npm test` *or* `npx playwright test` | Runs the test suite headlessly across configured environments. |
| **Run UI Mode** | `npx playwright test --ui` | Opens the interactive Playwright UI runner (great for step-by-step debug and inspection). |
| **Debug Mode** | `npx playwright test --debug` | Opens the Playwright Inspector, allowing you to step through actions line by line. |
| **Headed Mode** | `npx playwright test --headed` | Shows the browser UI while running tests. |
| **Targeted Spec** | `npx playwright test tests/import-project.spec.ts` | Runs only the specified test file. |
| **Show HTML Report** | `npx playwright show-report` | Opens the HTML report generated by the last test run. |

## Project Structure

```
e2e-tests/
├── tests/
│   ├── import-project.spec.ts     # Main E2E spec verifying Project Import / Manual Creation
│   └── utils/
│       └── excelGenerator.ts      # Utility to generate valid/invalid Excel files in memory
├── playwright.config.ts           # Playwright framework and runner configuration
├── package.json                   # Project dependencies and script definitions
├── test_plan.xlsx                 # The master test case definitions workbook
└── tc-075-debug.png               # Reference screenshot used for debugging
```

### 1. playwright.config.ts
Configures the test runner environment:
- **Base URL**: Set to `https://dev.nexus-procure.pages.dev`
- **Target Browser**: Chromium (configured under projects)
- **Parallelism**: Runs tests fully parallel (`fullyParallel: true`)
- **Reporter**: Generates a standard `html` test execution report
- **Traces**: Records traces on the first retry of any failed tests

### 2. tests/import-project.spec.ts
A comprehensive spec containing over 40 distinct test cases checking the **Import Project** workspace functionality. It is split into 12 functional suites matching the Excel test plan.

### 3. tests/utils/excelGenerator.ts
A helper function that builds custom, in-memory spreadsheet buffers matching both the **LessPaper** and **Fiori** templates. This eliminates the need to maintain static Excel file assets, letting tests programmatically feed specific cell values, missing fields, or duplicate lists directly to file upload inputs.

## Detailed Test Coverage

The test cases in `import-project.spec.ts` map directly to the `test_plan.xlsx` workbook:

### 1. Auth & Access Control
- `TC-001`: Authorized users (e.g., `document_staff`) can access the Import Project route.
- `TC-002`: Unauthorized users (e.g., `guest`) are denied access and redirected.

### 2. File Upload Validations
- `TC-003`: Rejects invalid file extensions (non-Excel files like `.txt`).
- `TC-004` & `TC-444`: Validates the row limit boundary (fails at 51 rows, passes at exactly 50 rows).

### 3. Parsing States & Errors
- `TC-005`: Verifies that the preview table renders when a valid file is parsed.
- `TC-006`: Shows a friendly error notice when parsing corrupted or non-Excel file contents.

### 4. LessPaper Bulk Import Flow
- `TC-007`: Enforces required column validations (missing names, budgets, methods, etc.).
- `TC-008`: Handles 409 database conflicts for pre-existing PR numbers.
- `TC-009`: Detects duplicate PR numbers within the uploaded spreadsheet rows.
- `TC-010`: Prevents non-positive budget values (e.g., negative budget values).
- `TC-011`: Enforces method-specific budget thresholds (e.g. `LT100K` limit ≤ 100,000 THB).
- `TC-012`: Rejects past delivery dates.
- `TC-013` & `TC-014`: Validates in-table row editing and deletion behaviors prior to submission.

### 5. Fiori Bulk Import Flow
- `TC-015`: Enforces required field logic specific to Fiori files.
- `TC-016`: Rejects pre-existing PR numbers in Fiori database imports.
- `TC-017`: Prevents duplicate PR numbers within the Fiori spreadsheet rows.
- `TC-018`: Autopopulates the default procurement method if empty.
- `TC-019` to `TC-022`: Validates in-table correction edit loops and row deletion.

### 6. Manual Form - User & Scoping Constraints
- `TC-023`: Locks department and unit fields for Unit Representatives who only oversee a single group (e.g., `facilities_rep`).
- `TC-024`: Allows selection across multiple owned units for Representatives mapping to multiple groups (e.g., `student_affairs_rep`).
- `TC-025`: Enables full organization selection for Procurement staff (e.g., `document_staff`).

### 7. Manual Form - Validations & Gating
- `TC-026`: Standard front-end validation triggers blocking empty submissions.
- `TC-027`: Handles duplicate PR database checks.
- `TC-028`: Enforces positive budgets.
- `TC-029`: Validates procurement type budget limits.

### 8. Manual Form - Budget Plan Selection
- `TC-030` to `TC-033` & `TC-424`: Validates details (activity name, cost center, amount) of fetched budget plans, auto-fills standard budgets on plan selection, and throws a warning when manually lowering the budget below the plan standard.

### 9. Manual Form - Delivery Date & Urgency Warnings
- `TC-034` & `TC-425`: Alerts users if delivery date is under the 15-day threshold, requiring an acknowledgement check in a confirmation dialog before proceeding.
- `TC-446`: Factors Thai public holidays dynamically (utilizing mocked calendars) to properly calculate net working days.

### 10. Success Flow, Routing & Navigation
- `TC-035`, `TC-037`, `TC-038`, `TC-040`, `TC-041`: Ensures a successful project creation redirects to the success page, auto-generates numbers prefixed with the target fiscal year (e.g., `2570/`), defaults project status to "ยังไม่ได้มอบหมาย", and redirects staff users to the general project pool.
- `TC-039`: Ensures Unit Representatives are redirected straight back to the manual form when clicking "สร้างโครงการเพิ่มเติม".

### 11. Templates
- `TC-042` & `TC-043`: Confirms file download triggers and validates the suggested filenames for LessPaper and Fiori templates.

### 12. Edge Cases
- `TC-448`: Prevents duplicate form submissions by immediately disabling buttons during active requests.
- `TC-449`: Validates partial bulk import rollbacks on API failures.
